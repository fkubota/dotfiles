# Claude グローバル設定

## ユーザープロファイル自動保存機能

**重要**: この機能は全セッションで自動的に有効です。

### 目的
会話を通じてユーザーについて学んだ情報を自動的に保存し、より良いサポートを提供する。

### 動作ルール

#### 1. 情報の自動検出と保存
会話の中で以下の情報を検出したら、**自動的に**対応するファイルに保存すること：

- **個人情報**: 年齢、名前、家族構成、住所、出身地など
  → `~/.claude/user-profile/personal.md` に保存

- **仕事関連**: 職業、職種、勤務先、プロジェクト、スキル、経験年数など
  → `~/.claude/user-profile/work.md` に保存

- **興味・嗜好**: 趣味、好きなもの、嫌いなもの、価値観、こだわりなど
  → `~/.claude/user-profile/preferences.md` に保存

- **技術スタック**: 使用している言語、フレームワーク、ツール、環境など
  → `~/.claude/user-profile/tech-stack.md` に保存

- **その他のコンテキスト**: 現在取り組んでいることや重要な背景情報
  → `~/.claude/user-profile/context.md` に保存

#### 2. 保存のタイミング
- ユーザーが明示的に情報を共有したとき（例：「私は30歳です」）
- 会話の文脈から情報が推測できるとき（例：「子供が...」→ 家族構成）
- 技術的な質問から使用技術がわかったとき（例：React の質問 → tech-stack）

#### 3. 保存時の注意
- **ユーザーに明示的に通知しない**: 自然に保存を行い、「保存しました」などの通知は不要
- **情報量の管理**: ファイルが大きくなりすぎないよう、古い情報は削除し、重複は統合し、適宜要約を行う
- **既存情報の更新**: 新しい情報が古い情報と矛盾する場合は、古い情報を更新または削除する
- **推測の場合は明記**: 確定情報と推測を区別する（例：「[推測]」タグ）
- **プライバシーに配慮**: センシティブな情報は慎重に扱う

#### 4. ファイルフォーマット
各ファイルは以下の形式で記録（簡潔に保つこと）：

```markdown
# [カテゴリ名]

最終更新: YYYY-MM-DD

## 基本情報
- 項目: 値 (更新: YYYY-MM-DD)

## 補足
- 重要な追加情報のみ
```

#### 5. 情報の整理と要約
- **定期的な見直し**: ファイルが50行を超えたら、古い情報を削除するか要約する
- **重複の削除**: 同じ情報が複数回記録されている場合は統合する
- **優先順位**: より新しく、より重要な情報を優先して保持する
- **コンテキスト管理**: ファイル全体を常に簡潔に保ち、読み込み時のコンテキスト消費を最小化する

#### 6. 初回セッション時の動作
プロファイルファイルが存在しない場合は、初回の会話で検出した情報から自動的に作成する。

#### 7. 既存情報の活用
セッション開始時に既存のプロファイルファイルを読み込み、ユーザーのことを理解した上で会話を進める（ただし、「あなたのプロファイルを読みました」などと明示的に言う必要はない）。

---

## 実装上の注意

- この機能は**バックグラウンドで動作**し、ユーザーの会話フローを妨げない
- 情報の保存は Edit または Write ツールを使用して行う
- ファイルが既に存在する場合は Read してから Edit で更新する
- 不確実な情報は保存しない、または [推測] タグを付ける
- `~/.claude/user-profile/` ディレクトリが存在しない場合は、最初に自動的に作成する
- **保存は控えめに**: 重要な情報のみを保存し、些細な情報は保存しない

---

## プライバシー

- すべての情報はローカルマシン (`~/.claude/user-profile/`) にのみ保存
- 外部サーバーへの送信は一切行わない
- ユーザーはいつでもファイルを確認・編集・削除できる
