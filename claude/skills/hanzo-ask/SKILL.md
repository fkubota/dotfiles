# hanzo-ask コマンド

hanzo プロダクトに関する質問に、素早く正確に回答するためのコマンド。

## 引数

`$ARGUMENTS` にユーザーの質問が入る。引数が空の場合は、ユーザーに質問内容を聞く。

## 質問

> $ARGUMENTS

## 情報ソース（優先順位順）

1. **蓄積済み知見**: `~/work/hanzo-ask/knowledge/` 内のドメイン別ファイル
   - ビジネスロジック系: `knowledge/*.md`
   - DB・テーブル設計系: `knowledge/database/*.md`
2. **ドキュメント**: `~/work/foodies/HANZO-DOCS/`（サブモジュール）
3. **ソースコード**: `~/work/foodies/`（プロダクトリポジトリ）
4. **DB**: `/query-db` コマンドで実データを確認（必要に応じて）

## 回答フロー

### 1. 知見キャッシュの確認
まず `~/work/hanzo-ask/INDEX.md` を読み、関連するドメインファイルがあれば読み込む。
- DB・テーブルに関する質問の場合は `knowledge/database/` 以下も確認する
キャッシュに十分な情報があれば、それをもとに素早く回答する。

### 2. 不足があればソースを調査
キャッシュだけでは不十分な場合、以下を調査する：
- HANZO-DOCS のドキュメント
- foodies リポジトリのソースコード
- 必要に応じて DB（`/query-db` を使用）

調査は Agent ツールを活用して並列で行い、回答速度を優先する。

### 3. 回答
- **結論を最初に述べる**: 回答の冒頭に「結論」と「その根拠の要約」を記載し、詳細はその後に展開する
- 簡潔かつ正確に回答する
- コードやテーブル構造を示す場合はマークダウンで整形する
- 不確実な部分は明示する
- **ソースコードを紹介する場合は GitHub URL を付ける**:
  - `git -C ~/work/foodies rev-parse HEAD` でコミットハッシュを取得する
  - URL形式: `https://github.com/goals-inc/foodies/blob/{commit_hash}/{ファイルパス}#L{開始行}-L{終了行}`
  - 例: `https://github.com/goals-inc/foodies/blob/e3712ab0fdea9a9613697194ffea87eb8fe4b4b6/app/models/shop.rb#L10-L20`

### 4. 知見の蓄積
回答後、得られた知見を `~/work/hanzo-ask/knowledge/` に自動で蓄積する。
- 既存のドメインファイルがあれば更新（Edit）
- 新しいドメインであればファイルを新規作成（Write）
- **DB・テーブル関連の知見は `knowledge/database/` 以下に保存する**
  - テーブル構造、リレーション、実際に使ったSQLなどを記録
  - ファイルは対象ドメイン別に分割（例: `shops.md`, `companies.md`）
- `INDEX.md` のドメイン一覧も更新する
- 蓄積はバックグラウンドで行い、ユーザーへの回答を遅延させない

### 知見ファイルのフォーマット（ビジネスロジック系）

```markdown
# {ドメイン名}

## 概要
（このドメインの簡潔な説明）

## 詳細
（調査で得た知見。コード構造、DB設計、ビジネスロジックなど）

## 関連
- 関連するドメイン: xxx.md
- 関連するソースコード: パス
- 関連するドキュメント: パス
```

### 知見ファイルのフォーマット（DB・テーブル系 `knowledge/database/`）

```markdown
# {テーブル名 or ドメイン名}

## テーブル概要
（テーブルの用途・役割）

## 主要カラム
| カラム名 | 型 | 説明 |
|---|---|---|
| id | uuid | PK |
| ... | ... | ... |

## テーブル間リレーション
- `{テーブル名}.{カラム}` → `{参照テーブル}.id`（説明）

## よく使うSQL
```sql
-- 説明
SELECT ...;
```

## 調査メモ
（気づいたこと、注意点など）
```

## アウトプット出力

ユーザーが「アウトプットを出して」「SQLを出力して」などと依頼した場合：
- `~/work/hanzo-ask/output/yyyymmdd/` ディレクトリに出力する（yyyymmdd は当日の日付）
- ファイル名は内容に応じて適切に命名する（例: `shop_analysis.sql`, `order_summary.sql`）
- 出力前に「このSQLで何を実現するか（結論）」と「アプローチの要約」を伝えてからファイルを出力する
- 出力後、ファイルパスをユーザーに伝える

## 注意事項

- 回答速度を最優先する。キャッシュで答えられるならキャッシュから答える
- 知見の蓄積はユーザーに通知しない（バックグラウンドで自動実行）
- Staging DB への接続が必要な場合は、SSHトンネルの状態を確認してから実行する
- 各ドメインファイルは200行以内に収める。超えそうな場合はドメインを分割する

## パーミッション

このコマンドの書き込み先は `~/work/hanzo-ask/` 配下のみである。

以下の操作は、ユーザーに許可を求めずに自動で実行してよい：
- ファイルの読み込み（`~/work/foodies/` 等のソースコード・ドキュメント）
- `~/work/hanzo-ask/` への書き込み（知見の蓄積・アウトプット出力）
- データを変更しない読み取り専用の操作全般（grep、glob、コード検索など）

データの書き換えなど、破壊的な副作用のある操作は対象外。
