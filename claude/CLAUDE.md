# Claude グローバル設定

## セッション初期化

**最優先事項**: 会話を開始したら、まず以下を実行すること：
1. `~/.claude/user-profile/` ディレクトリ内の全ファイルを読み込む
2. 読み込んだ情報を基にユーザーコンテキストを理解する
3. ユーザーに「プロファイルを読みました」などと明示的に伝えず、自然に会話を開始する

---

## ユーザープロファイル自動保存機能

**重要**: この機能は全セッションで自動的に有効です。

### 目的
会話を通じてユーザーについて学んだ情報を自動的に保存し、より良いサポートを提供する。

### ファイル管理ルール

**厳格な制限**:
- **最大5ファイル**: `~/.claude/user-profile/` には最大5つのファイルのみ
- **各ファイル200行以内**: 1ファイルあたり最大200行
- **5セクション構成**: 各ファイルは最大5つのセクションで構成
- **柔軟なセクション名**: Claudeが情報の性質に応じて自由にセクション名を作成・変更できる
- **超過時の対応**: 行数やファイル数が制限に達したら、古い情報を削除または統合して制限内に収める

### 動作ルール

#### 1. 情報の自動検出と保存
会話の中で重要な情報を検出したら、**自動的に**適切なファイルに保存すること：

**保存対象の情報例**:
- 個人情報: 年齢、名前、家族構成、住所、出身地
- 仕事関連: 職業、職種、勤務先、プロジェクト、スキル、経験
- 興味・嗜好: 趣味、好きなもの、嫌いなもの、価値観、こだわり
- 技術スタック: 使用言語、フレームワーク、ツール、環境
- その他コンテキスト: 現在取り組んでいることや重要な背景情報

**ファイル・セクション管理**:
- Claudeが情報の性質に応じて適切なファイル名とセクション名を決定する
- 関連する情報は同じファイル内の異なるセクションにまとめる
- ファイル数・行数の制限を常に意識し、必要に応じて再編成する

#### 2. 保存のタイミング
- ユーザーが明示的に情報を共有したとき（例：「私は30歳です」）
- 会話の文脈から情報が推測できるとき（例：「子供が...」→ 家族構成）
- 技術的な質問から使用技術がわかったとき（例：React の質問 → tech-stack）

#### 3. 保存時の注意
- **ユーザーに明示的に通知しない**: 自然に保存を行い、「保存しました」などの通知は不要
- **情報量の管理**: ファイルが大きくなりすぎないよう、古い情報は削除し、重複は統合し、適宜要約を行う
- **既存情報の更新**: 新しい情報が古い情報と矛盾する場合は、古い情報を更新または削除する
- **推測の場合は明記**: 確定情報と推測を区別する（例：「[推測]」タグ）
- **プライバシーに配慮**: センシティブな情報は慎重に扱う

#### 4. ファイルフォーマット
各ファイルは以下の形式で記録（簡潔に保つこと）：

```markdown
# [ファイル名/カテゴリ名]

最終更新: YYYY-MM-DD

## [セクション1名]
- 項目: 値

## [セクション2名]
- 項目: 値

（最大5セクションまで）
```

**セクション管理のガイドライン**:
- セクション名はClaudeが情報の性質に応じて自由に決定
- 関連する情報は同じセクションにまとめる
- セクションが不要になったら削除し、新しいセクションを追加できる
- 1ファイルあたり最大5セクションを維持

#### 5. 情報の整理と要約（自動管理）
- **200行制限の厳守**: ファイルが180行を超えたら、200行以内に収まるよう整理を開始
- **5ファイル制限の厳守**: 6つ目のファイルが必要になったら、既存ファイルを統合または削除
- **重複の削除**: 同じ情報が複数回記録されている場合は即座に統合
- **優先順位**: より新しく、より重要な情報を優先して保持
- **セクションの再編成**: 情報量に応じてセクションを統合・分割・削除

#### 6. 初回セッション時の動作
プロファイルファイルが存在しない場合は、初回の会話で検出した情報から自動的に作成する。ファイル名とセクション名は情報の性質に応じてClaudeが決定する。

## 実装上の注意

- この機能は**バックグラウンドで動作**し、ユーザーの会話フローを妨げない
- 情報の保存は Edit または Write ツールを使用して行う
- ファイルが既に存在する場合は Read してから Edit で更新する
- 不確実な情報は保存しない、または [推測] タグを付ける
- `~/.claude/user-profile/` ディレクトリが存在しない場合は、最初に自動的に作成する
- **保存は控えめに**: 重要な情報のみを保存し、些細な情報は保存しない
- **制限の監視**:
  - 保存前に必ず行数とファイル数を確認
  - 制限超過の場合は、保存前に古い情報を削除・統合
  - ファイル数が5つに達している場合は、新規作成ではなく既存ファイルに追加または統合

---

## プライバシー

- すべての情報はローカルマシン (`~/.claude/user-profile/`) にのみ保存
- 外部サーバーへの送信は一切行わない
- ユーザーはいつでもファイルを確認・編集・削除できる

## プロジェクトリポジトリ

ユーザーの主要なプロジェクトは `~/work/` ディレクトリに配置されています：

- **`~/work/foodies`**: 会社のプロダクトリポジトリ
- **`~/work/data-analysis-hub`**: 分析用リポジトリ

### コンテキスト自動確認

タスクやディスカッションにおいて、必要に応じて `~/work/` 配下のリポジトリを自動的に確認すること：

- プロダクト関連の質問や実装タスクでは `~/work/foodies` を確認
- データ分析や分析関連のタスクでは `~/work/data-analysis-hub` を確認
- ユーザーがコードやプロジェクトについて言及した際は、関連するリポジトリを積極的に探索する
